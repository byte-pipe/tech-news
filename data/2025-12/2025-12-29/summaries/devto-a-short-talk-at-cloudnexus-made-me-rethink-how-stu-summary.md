# A short talk at CloudNexus made me rethink how students are using AI - DEV Community

# Summary of the Article "A Short Talk at CloudNexus: Rethinking How Students are Using AI"

## Key Points

* The speaker discussed their experience as a speaker at an AWS Cloud Club QAU event on cloud technologies and AI, focusing specifically on agentic AI.
* Despite being surprised by students' confidence in using AI tools effectively, the speaker revealed several concerns about how they're being used.

## Main Ideas

* Students are not only "using" AI for code generation but also other purposes such as understanding concepts, writing or structuring content, and debugging, leading to challenges down the line.
* Real-world AI development requires more than just using code-focused tools; it demands cloud-aware guidance, best practices, security-conscious suggestions, and agent-based systems that work in isolation.

## How Students are Using AI

* ChatGPT: mainly for code generation, not understanding or learning concepts.
* Coding copilots (e.g., Cursor, Amazon Q): useful for development work by providing context and building on existing data; does not live outside the workflow.
* Cloud AI (including AWS Bedrockstart): enables agent-based systems that can be used to automate outcomes, but requires infrastructure, scale, and security.

## Takeaway

* The speaker emphasizes that AI should augment human capabilities, rather than replace them, and suggests using various levels of AI depending on the situation:
	+ Learn from ChatGPT.
	+ Build with coding copilots for development work.
	+ Scale with cloud AI to achieve automation goals.

The speaker concluded by acknowledging that students do not necessarily need "AI experts" but can benefit from basic understanding of various aspects of AI, including its limitations and applications.
