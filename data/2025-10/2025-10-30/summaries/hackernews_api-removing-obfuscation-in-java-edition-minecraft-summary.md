# Removing obfuscation in Java Edition | Minecraft

**Removing Obfuscation in Java Edition**

**What this means for modding enthusiasts:**
The Minecraft community can now focus on creating mods without the need to unravel complex code.

**An update of Java Edition's obfuscation practices**
Java Edition has been using obfuscation (hiding parts of its code) since its release. This meant that developers could not easily see the source code and had a harder time making modifications.

**Introduction of "obfuscation mappings" in 2019**

These mappings provided a way for modders to interpret obfuscated terms, but it still became a tedious process. To simplify this, Java Edition is now removing obfuscation altogether.

**Changes effective from Mounts of Mayhem launch**
As of the complete Mounts of Mayhem launch snapshot (October 29, 2025), all future builds will have their code names included by default with variable and other names to facilitate modding. This marks a significant improvement in tool support for existing modding tools.

**Modder impact on change:**

**Preparing for the transition**
A new un-obfuscated "experimental release" version of Java Edition will be available alongside obfuscated versions at this point.

**Expected completion**
The final version with no obfuscation will follow shortly after.

**Benefits to modmers**

* Easier code modification
* De-bugging becomes more straightforward
* Crashes logs are readable
