# Angular's Testing Revolution: Vitest, Fake Timers & Testronaut - DEV Community

## Angular's Testing Revolution: Vitest, Fake Timers, and Testronaut

### Key Points:

* Vitest is a new default testing framework in Angular 21, replacing Karma.
* Async testing will be addressed in specific zoneless Angular applications when using Vitest.
* Testronaut will debut as the community version of Playwright Component Testing for Angular.

## Vitest Overview

The most important thing about Vitest is that it brings a powerful API with a rich ecosystem focused on TypeScript, ES6 support, and browser mode tests.

### Features:

* Powerful API
* Rich, forward-looking ecosystem
* TypeScript-first and supports ESM
* Browser mode testing

### Decision Making

Vitest was chosen because it addresses the uncertainty of Jasmine/Karma being replaced with Vitest in the browser environment. The Angular team wanted to move forward but not force everyone to rewrite their tests.

## Conclusion

Angular wants to be part of the broader JavaScript ecosystem and has decided on Vite for its implementation as part of the new testing framework, Vitest.

### Key Statistics:

* Vitest will become the default testing framework in Angular 21.
* Async testing is addressed in zoneless applications using Vitest when applicable.
* Testronaut will debut as the community version of Playwright Component Testing for Android.
