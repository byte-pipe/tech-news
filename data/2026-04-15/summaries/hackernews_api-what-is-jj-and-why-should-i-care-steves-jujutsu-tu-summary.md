# Summary of “What is jj and why should I care?”

## Overview
- **jj** is the command‑line interface for **Jujutsu**, a distributed version control system (DVCS).
- The tutorial assumes readers are familiar with Git and are considering a switch from Git to jj.

## Key Advantages
- **Simplicity and Power**: Claims to be both simpler/easier to use than Git while offering greater power.
- **Best of Both Worlds**: Combines strengths of Git and Mercurial (hg) into a new, yet familiar workflow.
- **Fewer Core Tools**: Reduces the number of essential commands, making them more powerful through cleaner integration.
- **Advanced Capabilities**: Offers sophisticated features that are difficult to achieve with Git alone.

## Compatibility and Adoption
- Provides a **Git‑compatible backend**, allowing you to use jj on existing repositories without forcing collaborators to change tools.
- Switching to jj does not jeopardize existing history; you can revert to Git at any time with no loss of data.
