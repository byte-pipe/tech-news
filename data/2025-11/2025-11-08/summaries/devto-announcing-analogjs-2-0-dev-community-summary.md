# Announcing AnalogJS 2.0 ⚡️ - DEV Community

**Angular AnalogJS 2.0 Release**

### Key Features and Improvements

The AnalogJS team is excited to announce the release of Angular AnalogJS 2.0, providing developers with a powerful meta-framework for building websites and applications. This release introduces significant improvements and new features.

*   **Expanded Support for Vite Ecosystem**: AnalogJS now offers better integration with Vite, a next-generation build tool.
*   **Better User Experience**: Features such as markdown page templates and API/server routes provide a streamlined development experience.
*   **Improved Deployment**: The release comes with enhanced server and deployment support for hybrid single-sided rendering (SSR) and single-page application state change tracking (SGS).
*   **Enhanced Filesystem-Based Routing**: This allows developers to simplify frontend routing.

**What's New**

Key features in the 2.0 release include content resources, smaller install bundle size optimizations, and Vite ecosystem upgrades.

#### Content Resources

Content resources are used to display lists of loaded content, providing better user experiences for various applications.

*   **Displaying List of Loaded Content**: Include the `contentFilesResource` class to retrieve and display a list of loaded content.
*   **Importance in App Development**

#### Server and Deployment

The release includes server and deployment support for hybrid single-sided rendering (SSR) and single-page application state change tracking (SGS).

*   **Implementation of Hybrid SSR/SGS**: Enable developers to integrate AnalogJS with their existing SSR or SGS setup.
*   **Better Control Over Your Application's State**: Enhanced server and deployment capabilities ensure seamless integration with larger-scale applications.

### Additional Improvements

#### Smaller Install Bundle Size Optimizations

The implementation of smaller install bundle size optimizations reduces the overall download size, allowing you to deploy your application more efficiently in various environments.

*   **Optimized Installation Process**: Use the `contentBundleSize` directive to optimize the installation process.
*   **Better Overall Deployment Experience**

#### New Component Examples and Resolvers

Examples of new components and resolvers are provided for better understanding of how to integrate AnalogJS into your application workflow.

*   **New Markdown Component Example**: A basic markdown component example showcases a modern implementation of content rendering within Angular applications.
*   **Improved Post Resolver and Meta Resolver Code Examples**: Get guidance on writing effective resolvers that work seamlessly with content resources and store configurations.
