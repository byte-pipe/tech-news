# Common Coding Mistakes at Every Level (And How to Fix Them) - DEV Community

# Understanding Coding Mistakes

Every developer experiences moments when they realize that a bug they've been hunting for hours was caused by a simple typo, such as missing semicolons or variable names.

**The Weight of Human Fallibility**

This feeling is common among coders, with even junior developers making mistakes. However, identifying and learning from these errors is crucial in software development.

## Mistakes are Just Patterns Waiting to be Recognized

Understanding why something goes wrong allows you to fix it. This process doesn't just happen in code; it can also occur when thinking critically about technical issues.

**Organizing Coding Mistakes for Later Reference**

### Part I: Beginner Mistakes (Or, "Welcome to the Thunderdome")

1. **Treating Error Messages Like Personal Attacks**
    * Feels like a personal attack
    * Intimidates with cryptic language and unfamiliar terms

2. **Ignoring Error Messages**
    * Panic or Google random things in code
    * Does not provide valuable information

### Fixed: Learning to Love Error Messages

* Read error messages carefully, looking for the most important information
* Locate:
    + Type of error (TypeError/SyntaxError/...)/File/Line number
    + Actual explanation of what went wrong
