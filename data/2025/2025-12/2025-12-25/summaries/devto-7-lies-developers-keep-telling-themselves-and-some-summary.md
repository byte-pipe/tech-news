# 7 Lies Developers Keep Telling Themselves (And Somehow Still Believe) - DEV Community

# Developer Lies to Tell Themselves

## Key Points:

* Developers often pretend that their code is perfect or will eventually be.
  * Lie: "Nobody Writes Clean Code"
* Many developers fall for the trap of saying it'll only take a moment.
  * Tip to avoid this lie: look for hidden traps and make sure they won't fall back into the same pattern.
* Developers often delay refactoring tasks until later, when it's too hard or convenient.
  * Pro tip: create refactor tasks that can be completed quickly, and add these to your backlog.
* Some developers downplay the importance of writing tests by claiming they're not necessary if no bugs are supposed to occur.
  * Lie/Comment for discussion: "It's just an edge case"
* Despite initial success with certain assumptions (e.g., frontend/backend), most bugs do show up everywhere.
  * Comment: "If it works on my machine, it'll work everywhere"

## Additional Points:

* Many developers struggle to separate their personal code opinions from professional ones.
* Developers often need reminders that working together doesn't mean everyone sees eye-to-eye or shares similar expertise.

# Conclusion

While these lies might seem trivial for the individual developer who reads them, they can significantly affect team productivity and overall project outcomes. By becoming more aware of these common developer lies, we can create a more collaborative work environment and improve code quality.
