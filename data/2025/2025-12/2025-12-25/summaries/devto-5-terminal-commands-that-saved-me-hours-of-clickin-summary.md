# 5 Basic Commands That Saved Me Hours of Clicking - DEV Community
When started coding, I was initially terrified of the terminal. The blank black screen with blinking green cursor looked like something out of The Matrix.

However, as I dived into more complex tools and projects, I realized that the GUI was feeling slow and outdated. To be fast, efficient, and understand what the computer was doing, I had to ditch the mouse and embrace the command line.

## The Top 5 Terminal Commands That Changed Everything
### 1. **cd (Plus Tab Completion)**

The Old Way: Open file explorer. Double-click "Documents". Double-click "Coding". Double-click "Python". ...

The Terminal Way:

* `cd ~/Doc[TAB]/Cod[TAB]/Py[TAB]/Proj[TAB]/Web[TAB]`: Autocompletes folder names with Tab key
* Enter fullscreen mode, exit fullscreen mode

Why I love it:
cd stands for "Change Directory." It provides a simple and effective way to navigate through folders.

### 2. **mkdir -p**

The Old Way: Set up new project structure. Right-click -> New Folder -> type...
The Terminal Way:

* `mkdir -p my-app/{src/components,assets,tests}`: Instantly creates main folder and subfolders
* Enter fullscreen mode, exit fullscreen mode

Why I love it:
mkdir means "Make Directory." With the -p flag (parent) and curly braces {}, it can create entire folders instantly.

### 3. **cp -r**

The Old Way: Backup project before making changes...
The Terminal Way:

* `cp -r my-cool-project my-cool-project-BACKUP`: Creates backup of the current folder
* Enter fullscreen mode, exit fullscreen mode

Why I love it:
cp is Cheetah Mode (C). It automatically copies files and folders recursively.

### Additional Tips for Speeding Up Your Coding Workflow:

* Use Tab key to complete directory names in commands.
* Experiment with different terminal configurations and settings to optimize your workflow.

Overall, mastering the 5 basic terminal commands saved me hours of clicking and frustration. By embracing the command line, I've become a more efficient and effective coder.
