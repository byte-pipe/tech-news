# Code Mode Doesn't Replace MCP (Here's What It Actually Does) - DEV Community

**Summary**

The article discusses MCP (Model Context Protocol), which has revolutionized agentic workflows by connecting agents to everyday apps. However, the experience is imperfect due to technical constraints, such as "tool bloat" or excessive extension use. Two experts, Nick Cooper and Bradley Axen, share their insights on these issues.

**Key Points**

* MCP introduces a new norm for connecting agents to apps, but faces challenges balancing tool power with technical limitations.
* "Tool bloat" happens when developers turn on too many extensions at once, causing degradation in performance, errors, and hallucinations.
* Dynamic extensions were initially implemented but remain hidden; current approach should be more accessible.

**Main Ideas**

1. The "tool bloat" problem is a significant challenge for MCP adoption, as it leads to user frustration, errors, and decreased performance.
2. Excessive extension use can cause degradation in system health, leading users to conclude the platform isn't ready for prime time.
3. Dynamic extensions aim to mitigate this issue by limiting tool utilization while allowing flexibility.
