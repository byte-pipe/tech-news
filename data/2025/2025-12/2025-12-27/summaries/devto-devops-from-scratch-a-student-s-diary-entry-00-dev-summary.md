# DevOps From Scratch: A Student's Journey

## Current State and Goals
As a Computer Science student at Queen Mary University London, I've been struggling to bridge the gap between my code working on localhost and understanding how it really lives in the real world. Despite having completed a dev.to course and creating a blog account, I feel like I'm not ready for DevOps skills.

## Why This Journey?
I want to document my journey from "standard coder" to DevOps Engineer, focusing on System Awareness and building experiences through hands-on learning. I strive to improve my code quality, efficiency, and adaptability using various tools and technologies.

## Breaking Down Challenges

*   **Imposter Syndrome**: Struggling with feelings of inadequacy and wanting to be recognized as a "real" DevOps Engineer.
*   **Focus on Code Over User Experience**: Focusing too much on learning new skills rather than considering the overall user experience.

### Key Takeaways

*   I intend to learn system-thinking concepts before diving into specific technologies.
*   My primary goal is to build practical knowledge, rather than relying only on theoretical resources.

**Roadmap (2025-2027)**

1.  **Linux Fundamentals**: Learning Linux command-line skills to understand how it's used in various environments.
2.  **Bash & Scripting**: Mastering bash scripting techniques to automate tedious tasks and enhance productivity.
3.  **Containerization**: Understanding Docker Concepts to create lightweight, reusable container images.
4.  **Cloud Infrastructure**: Navigating AWS without significant financial surprises using tools and practices.
5.  **CI/CD**: Automating every step of the application lifecycle from testing to deployment.

### The "User POV" Mindset

*   I will strive to design software users can interact with as if they're installing the application for the first time.
*   By focusing on user experiences, I aim to create more enjoyable and efficient working environments.
