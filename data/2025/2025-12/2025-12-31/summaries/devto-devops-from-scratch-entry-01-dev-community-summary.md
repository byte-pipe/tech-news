## File System Hierarchy System (FHS)
Linux operates on a file system hierarchy standard that defines the structure of files in a Linux-based system and their relationships. Here's an overview:

* **/bin & /usr/bin**: Executable tools like ls, cd, and python.
* **/etc**: Configuration files for DevOps; where one can change how web servers behave to configure server behavior.
* **/var/log**: Logs of a system that happens when the system fails or crashes.
* **/tmp**: Temporary files.

## Command-Line Utilities "The Lego" Logic
Linux understands and processes complex commands using pipes (-|), redirection (>>), and combined command-line utilities. These commands use pipes to connect one command's output with the input from another.

Key Concepts:
- Pipes: Output goes first, then it is appended or redirected.
- Redirecting: The destination for data flow control at which command line tool output could go in a file - used also when doing system administration tasks and configuring files.
