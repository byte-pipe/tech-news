# How I Made an MCP Server That Saves Me an Hour per Week - DEV Community

### Simplifying Podcast Publishing with MCP Servers and AI Agents

To streamline our podcast's publishing process, I built an MCP server that automates tasks related to publishing, including saving engineers time per week.

## Key Steps in Our Existing Publishing Workflow:

*   **Audio File Upload**: Users upload their episodes to the platform.
*   **Metadata Filling**: A web-based interface allows users to fill out metadata fields (show notes, tags).
*   **Code Update**: The updated repository must be pushed to our publishing platform using a pull request.

## Leveraging AI for Automating Tasks

I saw an opportunity for AI-powered automation in summarizing content and generating drafts of publishing artifacts (show notes, social media posts).

### An AI-Based Solution:

Using Python, I developed a script that automatically converts audio files to transcripts. It also generates drafts of show notes and social media posts from the transcript.

## Benefits of AI-Powered Solutions for Podcasters

*   **Save Time**: The script reduces the time spent on summarization tasks.
*   **Streamlined Workflow**: Automation simplifies the publishing process, minimizing manual steps and effort required.

By automating these tedious tasks using an MCP server and leveraging AI, engineers can save time, which is a critical aspect of our Accelerate AI with Cloud Run workshops. We're excited to explore new possibilities for podcast automation and further integrating AI with MCP servers to simplify and speed up our publishing process.
