# Node Version Manager (nvm) - POSIX-Compliant Bash Script

## Overview
The `nvm` is a Node.js version manager that allows users to easily switch between different versions of the Node.js runtime. This script provides a command-line interface to manage multiple active versions.

## Key Features

* **POSIX compliance**: Ensures consistency and predictability across various platforms.
* **Login required**: Notifications are only visible to logged-in users.
* **Simple, consistent API**: Provides a unified interface for managing different node.js versions.

## Usage
To use this script, simply run it as a bash script and sign in to access changes in notification settings.

The user will be able to verify login by viewing their Fork count.

### Example Use Case

`nvm.sh` is usually located at `/usr/local/nvsh`. Once installed, users can switch between different node.js versions using the following command:

```bash
source /usr/local/nvsh/bin.nvm # Switch to default branch 9.6k (latest)
```

### Branch Management

Users can list all available branches by executing:

```bash
nvm branches # List all available branches
```

To create a new branch, use the following command:

```bash
nvm branches <new-branch-name> # Create a new branch with name "<new-branch-name>"
```
