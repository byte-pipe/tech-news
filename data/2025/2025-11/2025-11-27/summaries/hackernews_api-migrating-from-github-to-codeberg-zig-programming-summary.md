# Migrating from GitHub to Codeberg ⚡ Zig Programming Language

# Migrating from GitHub to Codeberg: A Summary

**Introduction**
The article discusses the decision by members of the Zig software foundation to migrate from GitHub to Codeberg. The process started 7 years ago due to concerns over the stability and performance of the service, particularly with regards to ActionS scheduler.

**Key Points**

* GitHub has been hosting the Zig language project since its inception in 2013.
* After Microsoft acquired GitHub, developers experienced significant issues with the platform, including slow speeds, buggy code, and lack of progress on ActionS's improved scheduling feature.
* As a result, the development team decided to switch to Codeberg, which is currently a separate hosting service offering more stability and performance.

**Migration Steps**

* The author made Ziglang/zig on GitHub read-only in early November 2025.
* The canonical origin/master branch of the main Zig project repository has been migrated to https://codeberg.org/ziglang/zig.git.

**Post-Migration Considerations**
The article also mentions that members of the development team are currently considering how to address a growing liability related to GitHub Sponsors, a revenue source for the project. The development team plans to offer equivalent perks through a new platform called Every.org starting the next month.
