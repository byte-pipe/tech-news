# 10 Developer Habits That Separate Good Programmers From Great Ones - DEV Community

## Separating Good Programmers from Great Ones: Key Habits in Code Development

When it comes to distinguishing between good programmers and great ones, a crucial distinction lies in daily habits rather than raw intelligence or knowledge.

### Habit 1: They Read Code Far More Than They Write It

Many junior developers struggle to keep up with the demands of reading code extensively. However, those who can read code deeply more often than write it demonstrate a significant advancement over their peers.

- **Key Insight:** The first habit separates good from great programmers. Great developers are voracious code readers, just as an author is passionate about crafting compelling novels.
- **Implication:** Those with this habit will likely excel in technical fields where understanding the writing of others is indispensable.

### Habit 2: They Write Clean Code

Once the desire to read is established, focusing on writing clean code takes precedence. This means striving for elegance and maintainability in software development, ensuring that every piece of code serves a specific purpose without unnecessary complexity or performance overhead.

- **Deeper Insight:** Writing good code requires discipline. It is not enough to merely understand how algorithms work; one must also learn about data structures and computational design patterns.

### Habit 3: They Review Code and Reflect on Patterns

Proactive thinking involves recognizing patterns in existing code and identifying opportunities for improvement. Regularly reviewing code to refine it can enhance understanding, efficiency, and scalability.

- **Subsurface Understanding:** This habit is crucial for making strides towards a more impactful technical career.

#### General Implications:

*   Developing good coding skills is about both theoretical knowledge and practical experience.
*   A focus on pattern recognition helps catch recurring errors and streamline processes.
*   It empowers writers to analyze complex solutions, break down them into components, and implement the best available methods.
*   As skills are refined through repetition of small coding tasks within a large scope,
*   there's a growing understanding of how each piece contributes to overall code quality.
*   Such habits lead programmers towards an increasingly professional demeanor in their work.

## Practice Discipline

Disciplines to aid in developing good programming habits and the growth that comes with them:

-   Writing small scripts or games periodically is beneficial for testing coding intuition.
-   Participating in online discussions on GitHub, Reddit's r/webdev, etc. helps find common reference points among developers
-   Implementing certain tools or resources daily to improve problem-solving efficiency
