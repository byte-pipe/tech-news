# We Should All Be Using Dependency Cooldowns

## Overview

Dependency cooldowns are a simple yet effective way to mitigate a large majority of open source supply chain attacks. The problem with most supply chain security is that it's often overhyped and focused on vendor vendors, rather than the actual vulnerabilities.

## Main Points

* Supply chain attack structure: malicious change introduced, compromised project has limited window of opportunity to cause damage.
* Key to success: individual projects implement dependency cooldowns (via tools like Dependabot) and packaging ecosystems invest in first-class support for cooldowns.
* The gap between compromising the project and introducing malware is too large, making it difficult for attackers to gain an advantage.

## Takeaway and Recommendations

* Individual projects should apply cooldowns to their dependencies.
* Packaging ecosystems should prioritize investing in first-class support for cooldowns directly in their package managers.
* Dependabot and Renovate can be used as tools to implement cooldowns.
