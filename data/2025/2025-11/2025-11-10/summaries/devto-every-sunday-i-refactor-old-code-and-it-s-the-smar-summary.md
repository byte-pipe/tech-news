# Every Sunday, I Refactor Old Code and It’s the Smartest Habit I’ve Ever Built - DEV Community

**Sunday Code Refactoring Ritual**

* **Starting Point:** No 5AM wake-ups, minimalist setup with Lo-fi beats
* **Goal:** Read old code to improve and make it better without fixing bugs or building new features
* **Habit:** Every Sunday, open laptop, refactor old code on existing projects

**The Process:**

1. Open a random project from months ago (e.g., a React app)
2. Clean up the code, removing console logs, duplicated functions, and unnecessary CSS
3. Learn something new about the codebase or its implementation
4. Make small improvements to readability, consistency, and overall quality

**Benefits:**

* Raw, honest progress, like a mirror to one's development journey
* Small tweaks lead to big wins and unexpected insights
* Connecting with past self through code helps in reflecting on growth and learning from mistakes

**Key Takeaways:**

* Refactoring is not rewriting everything, but rather improving existing code
* Consistency matters more than complexity when it comes to readability
* Growth is about making small improvements over time, not a destination

**Limitation:**

* Never spend more than 2 hours on refactoring and still wants to look forward to next Sunday's quiet moment of reflection

**Advice:**
If feeling stuck or unmotivated by new learning every day, try focusing on "connection" in the original advice instead
