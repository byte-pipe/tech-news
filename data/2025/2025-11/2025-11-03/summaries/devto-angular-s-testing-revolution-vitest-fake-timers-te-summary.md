# Angular's Testing Revolution: Vitest, Fake Timers & Testronaut - DEV Community

**Angular Testing Revolution with Vitest 21**

The article discusses the upcoming launch of Angular 21, which will also introduce Vitest as its new default testing framework. This change brings excitement and relief to the Angular community, allowing developers to switch to a more powerful and TypeScript-first alternative.

**Key Points:**

* Vitest is set to be available as the new default testing framework in Angular 21.
* The decision was made due to the need for a unified testing environment that can run tests in the browser.
* Vitest's powerful API, including poll feature and async testing support, has been improved since its introduction in version 14.

**Testing Framework Features:**

* Polling Feature: allows tests to wait for asynchronous tasks to end before considering successful execution
* Async Testing Support: enables testing of asynchronous code using Promises or native callbacks

**Testronaut's Role:**

* Testronaut is the community version of Playwright Component Testing for Angular.
* It will be available alongside Vitest and support all existing tests.

**Benefits:**

* unified testing environment for Angular applications
* more powerful API with improved async testing support
* TypeScript-first and fully supports ES modules (ESM)
* browser mode allows tests to run in a real browser environment

The article concludes by noting that this change is not just about Vitest, but also about ensuring the community is equipped to take advantage of new technologies. The inclusion of Testronaut as a community version of Playwright Component Testing for Angular further enhances the development experience.
