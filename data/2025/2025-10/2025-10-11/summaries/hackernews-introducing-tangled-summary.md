# introducing tangled

**Tangled's Decentralized Collaboration Platform Overview**
=================================================================

Tangled is a new social-enabled Git collaboration platform built on top of the AT Protocol. It aims to provide developers with complete ownership of their code, open-source communities can self-govern freely, and coding should be social and fun.

**Decentralized Code Collaboration Models**
---------------------------------------------

Tangled adopts two popular models: ActivityPub's federated model and Radicle's entirely P2P model. Its approach combines the best features from both worlds by using a custom protocol called "knots." Knots are lightweight, headless servers allowing users to host Git repositories with ease.

**Key Features**

*   Managed knots for self-hosting on small-scale systems (Raspberry Pi) or larger community-scaled setups
*   Customized App View system showcasing interconnected replicas across all connected nodes
*   Seamless collaboration and contributions through a consolidated "view" of the entire network

**Core Tenets**
-----------------

1.  **Ownership of Data**: Users maintain complete control over their data, ensuring data security and integrity.
2.  **Low Barrier to Entry**: The platform makes it easy for newcomers to join and start contributing code without extensive technical expertise.
3.  **No Compromise on User Experience**: Tangled prioritizes ease of use by minimizing distractions and focusing on creating a seamless collaboration experience.
