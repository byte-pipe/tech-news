# Angular Testing Revolution: Vitest and the Future of Testing
===========================================================

## Introduction

The Angular team has been experiencing uncertainty surrounding testing frameworks, particularly with regards to Signal Forms. However, two weeks away from v21, a major update is coming - the advent of Vitest as the new default testing framework.

## Vitest Overview

Vitest is a TypeScript-first, ES modules-based testing framework designed by Google. It offers a powerful API and provides various features such as polling, async testing, and mock objects.

## Testing Framework Comparison

For a long time, Angular developers used other testing frameworks like Jasmine and Jest, but Vite was still not adopted. The introduction of Vitest addresses this gap to provide a seamless transition for the Angular ecosystem.

## Key Points

*   Vitest brings forward-thinking features such as async testing and full browser mode
*   Uses Vite under the hood and is fully TypeScript-first with ESM support
*   Includes browser mode, making tests run in a real browser environment
*   Powerful API with polling feature and mock objects
*   Supports asynchronous testing and exit polling

## Conclusion

The development of Vitest marks an important step towards adopting better testing practices within the Angular ecosystem. This new framework will simplify testing for both old projects and new ones, ultimately helping to increase compatibility between teams using different tools like Karma and Jest.

# Future Plans

As v21 approaches, users can expect improved usability, additional libraries support, and a more cohesive experience across various testing scenarios.
