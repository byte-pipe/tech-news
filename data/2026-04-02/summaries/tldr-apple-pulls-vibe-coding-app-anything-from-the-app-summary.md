# Apple steps up crackdown on vibe coding apps, pulls ‘Anything’ from the App Store

## Background
- Apple recently blocked updates to vibe‑coding platforms such as Replit and Vibecode, saying they breached long‑standing App Store rules that forbid apps from running code that alters their own or other apps’ behavior.
- In a statement to 9to5Mac, Apple clarified that the problem is not “vibe coding apps” in general but apps that violate specific sections of its policies:
  - **Section 2.5.2 of the App Review Guidelines** – requires apps to be self‑contained, prohibits reading or writing outside the app’s container, and bans downloading, installing, or executing code that changes features or functionality, except for limited educational cases where the source code remains fully viewable and editable.
  - **Section 3.3.1(B) of the Developer Program License** – permits downloaded interpreted code only if it does not change the primary purpose of the application.
- Apple’s stance: it allows tools that help users build other apps, but it disallows apps that can modify their own behavior by generating and running code outside the App Store review process.

## Removal of ‘Anything’
- The Information reported that Apple pulled the vibe‑coding app **‘Anything’** from the App Store, citing a breach of guideline 2.5.2.
- Developer **Dhruv Amin** tried to submit an update that would let users preview their vibe‑coded apps in a web browser instead of inside the Anything app. Apple rejected the submission and subsequently removed the app entirely.
- Apple did not respond to requests for comment. As of now, other vibe‑coding apps continue to be available on the App Store.
