# App Store | Age of Agent – Summary

## Background
- The original iPhone launch (2007) relied on web apps; the App Store was introduced in 2008 to provide native apps, discovery, distribution, trust, and payment.
- This bundled model generated hundreds of billions of dollars but was designed for human interaction (browsing, tapping, swiping).

## Agents Need APIs, Not App Stores
- AI agents fulfill user requests (e.g., “book a table”) by calling APIs directly, bypassing app downloads and GUIs.
- In late 2024 Anthropic released MCP (Model Context Protocol), a universal interface for agents to connect to any service—likened to “USB‑C for AI.”
- OpenAI adopted MCP in March 2025, followed by Google; the protocol was later donated to a neutral foundation, making the connection layer open and non‑monopolizable.

## Unbundling the App Store into Three Layers
| Layer | Description | Current Economic Outlook |
|-------|-------------|---------------------------|
| 1. Connection (MCP) | Open protocol that enables agents to reach services. | Commodity plumbing; no direct revenue. |
| 2. Discovery | Determines which service an agent selects (e.g., Uber vs. Lyft). | High‑value battleground; control of ranking algorithms yields massive power and potential revenue, similar to Google’s search‑ad model but with near‑100 % conversion. |
| 3. Payment | Aggregates micro‑transactions across services handled by agents. | Greenfield opportunity; likely competitive and low‑margin because of open connections and portable data. |

## Apple’s Real Lesson: Payment Layer Dominance
- Apple’s profit came mainly from forcing all in‑app purchases through its 30 % payment system, not from discovery.
- Lock‑in via hardware and ecosystem (iPhone, iCloud, iMessage, etc.) gave Apple the leverage to impose high fees.

## Can AI Platforms Replicate Apple’s Model?
- The agent era lacks Apple‑style lock‑in: hardware is not exclusive, MCP works across all agents, data is portable, and model competition is fierce.
- High payment fees would drive users to rival agents that can access the same services via the same MCP servers.
- Consequently, the payment layer is expected to be competitive and low‑margin, while discovery becomes the primary monetization target.

## Future Timeline and Key Questions
- More services will build MCP adapters; agent tool ecosystems will expand.
- Agents will evolve from user‑specified service choices to autonomous selection, making “who decides the ranking” a central commercial and regulatory issue (e.g., EU vs. Google Shopping).
- As agents handle larger daily transaction volumes, a unified payment infrastructure will emerge; the entity controlling it will effectively “mint” the agent economy.

**Bottom line:** The App Store solved distribution for a human‑centric platform. In the AI‑agent era, the open connection layer (MCP) removes monopoly potential, shifting value to discovery and, to a lesser extent, payment. Monitoring the evolution of ranking algorithms and payment aggregators will be crucial for understanding the next platform economy.
