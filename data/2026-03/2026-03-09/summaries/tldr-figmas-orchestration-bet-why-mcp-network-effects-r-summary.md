# Figma’s orchestration bet: Why MCP network effects redefine software defensibility

## Overview of the SaaSpocalypse
- Wall Street is repricing SaaS amid AI‑driven multiple compression, dubbed the $285 billion “SaaSpocalypse.”
- The market is treating all SaaS as vulnerable to large language model (LLM) disruption, applying blanket discounts across sectors.
- The author argues this is a sorting event, not random destruction, and that the right filter is missing.

## Shift from seat‑based growth to orchestration
- Traditional SaaS 2.0 model: build the best tool in a vertical, expand seats, retain customers, layer features, and grow ARR.
- In an “agentic” AI world, defensibility moves from feature polish to control of workflow flow.
- Owning a node in the orchestration graph (the network that connects AI agents to tools) becomes the new moat.

## Figma’s recent moves illustrate the new model
- On Feb 17, Figma and Anthropic launched “Code to Canvas,” letting Claude Code push UI objects directly into Figma as editable layers.
- This creates a bidirectional bridge: Figma’s Dev Mode MCP Server already pushes design tokens to AI agents, and now code can flow back into the design canvas.
- The integration runs on MCP, an emerging open protocol likened to the USB‑C of AI orchestration, already adopted by Claude, ChatGPT, Gemini, Copilot, Cursor, and VS Code.

## Routing density as the new moat
- Defensibility is now about **semantic density**: how richly a tool encodes workflow context.
- Thin layers (e.g., simple task managers) can be recreated by agents; dense layers (e.g., full design systems) lose fidelity when bypassed.
- Figma has become a shared design system across product, engineering, marketing, and non‑designers:
  - ~60 % of Figma Make files are created by non‑designers.
  - >75 % of customers use multiple Figma products.
- Agents that generate UI need the system context (tokens, constraints, component libraries) stored in Figma; routing around it degrades output.

## Supporting data
- Figma Make usage rose 70 % quarter‑over‑quarter.
- Enterprise customers are building designs weekly.
- GitHub highlighted managing >7,400 design tokens through Figma’s MCP integration, signaling validation from the largest code‑hosting platform.

## Risks and value compounding
- MCP’s openness means any design tool could implement the protocol, reducing exclusivity at the endpoint level.
- However, nodes gain value through accumulated context, workflow breadth, and cross‑functional design gravity—assets that cannot be replicated instantly.

## Market mis‑sorting
- Current market discounts are based on broad categories (horizontal vs. vertical, security vs. productivity, etc.).
- The author proposes a more precise filter:
  - Does the company control semantically dense workflow context?
  - Do AI agents lose fidelity when bypassing it?
  - Is the company building bidirectional bridges into the agent ecosystem?
  - Does usage compound as agent activity grows?
- Companies meeting these criteria may be reinforced, not disrupted, by AI.

## Implications for SaaS valuation
- Figma’s market cap fell from $70 billion at IPO to ~$12 billion, reflecting a shift from pricing for inevitability to pricing for perceived obsolescence.
- The author argues the correct framing is an architectural transition, not death.
- Future SaaS winners will be those whose workflow context cannot be routed around, not those with the most features.

## Takeaway
- The “SaaSpocalypse” is a capital‑markets reset to a new technical reality.
- Seat‑based metrics matter less; routing dependency and workflow centrality matter more.
- Architectural positioning—controlling dense, bidirectional orchestration nodes—will define the next wave of defensible SaaS businesses.
