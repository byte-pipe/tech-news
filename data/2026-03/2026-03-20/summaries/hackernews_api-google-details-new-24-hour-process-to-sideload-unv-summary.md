# Google details new 24-hour process to sideload unverified Android apps - Ars Technica

# Android Security Update: 24-Hour Verification Process Introduced by Google

**Overview**
Google plans to introduce a new 24-hour process for sideloading unverified Android apps starting in September 2026. This change aims to address malware issues across the entire device ecosystem.

**Key Points**

* **Restrictions on Sideloading**: Android phones will only install apps from verified developers, requiring identification and signing keys upload from outside Google Play.
* **Procedural Verification**: Users must enable developer settings and manually verify app installation by entering a device unlock pin/password before proceeding to sideload unverified packages.
* **Countdown Period**: A 24-hour countdown is implemented to combat social engineering attacks; users can bypass verification but not spur-of-the-moment.

**Process**
The procedure involves:

1. Enabling Developer Options (tapping software build number in About Phone seven times) and accessing "Allow Unverified Packages".
2. Filling the unverified package box by flipping the toggle, confirming, entering device unlock pin/password, restarting the phone, and waiting 24 hours.
3. Scroll past warnings to allow installation of unverified packages; accept terms and confirm understanding.
4. Finally, tap Install anyway in the package manager, bypassing verification.

**Concerns**
The long countdown period (24 hours) might be impractical for users who want direct access to sideloading apps without Google's intervention.
