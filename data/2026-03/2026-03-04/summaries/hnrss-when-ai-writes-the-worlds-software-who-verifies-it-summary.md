# When AI Writes the World's Software, Who Verifies It?

**When AI Writes the World's Software, Who Verifies It?**

**Introduction**

Code Metal recently raised $125 million to rewrite defense industry code using AI. Google, Microsoft, and Toyota are also investing in AI-generated code. By 2030, Microsoft predicts that 95% of all code will be AI-generated.

**TheRewriting of the World's Software**

Anthropic recently built a 100,000-line computer compiler using parallel AI agents in two weeks for under $20,000. The result can now produce large-scale software at astonishing speed. However, it is unclear who verifies the correctness of this code.

**Challenges and Limitations**

AI-generated code often fails basic security tests, leaving human reviewers to fix the issues. Even experts like Andrej Karpathy, author of "Accept All," acknowledge that the quality of AI-generated code is not yet satisfactory. When faced with errors, reviewers often rely on code they have hand-coded.

**Real-World Examples**

A single bug in OpenSSL led to the compromise of millions of users' private communications for two years. AI-generated code is now being used at a scale that surpasses the combined effort of code review, testing, and manual inspection. Even seemingly polished AI-generated work can require someone downstream to fix it.

**The Threat of AI-Generated Vulnerabilities**

AI-generated code can create a new supply chain with significant risks. Adversaries can poison training data or compromise a model's API, injecting vulnerabilities into systems they touch. These risks are not hypothetical and already exist in the form of supply chain attacks.

**Conclusion**

As AI-generated code becomes more prevalent, our ability to verify its correctness becomes increasingly uncertain. The writing and thinking done by AI can outsourceto engineers, leaving them struggling to understand their software systems. The threat of AI-generated vulnerabilities highlights the need for continued innovation and improvement in code review and testing.
