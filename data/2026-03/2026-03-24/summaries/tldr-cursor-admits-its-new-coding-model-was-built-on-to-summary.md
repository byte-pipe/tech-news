# Cursor admits its new coding model was built on top of Moonshot AI’s Kimi

## Main points
- Cursor released Composer 2, marketing it as “frontier‑level coding intelligence.”
- An X user (Fynnsoon) claimed Composer 2 is essentially “Kimi 2.5” with added reinforcement learning, pointing to code that identifies Kimi as the underlying model.
- Cursor’s VP of developer education, Lee Robinson, confirmed Composer 2 started from an open‑source base (Kimi 2.5) but stated only about ¼ of the compute for the final model came from the base; the rest was from Cursor’s own training, leading to different benchmark performance.
- Robinson asserted that Cursor’s use of Kimi complies with the model’s license. The Kimi X account later congratulated Cursor, describing the relationship as an “authorized commercial partnership” with Fireworks AI.
- Cursor co‑founder Aman Sanger acknowledged the omission of Kimi in the launch blog and promised to correct it for future releases.

## Licensing and partnership details
- Kimi’s license permits commercial use, which Cursor claims to respect.
- The Kimi X account highlighted the integration as part of an “open model ecosystem” supported by Cursor’s additional pre‑training and high‑compute RL training.

## Industry context
- The revelation is notable because Cursor is a well‑funded U.S. startup (recent $2.3 B funding round, $29.3 B valuation, >$2 B annualized revenue) and had not mentioned any Chinese collaboration in its announcement.
- Building on a Chinese model amid the U.S.–China AI “arms race” can be politically sensitive, echoing past concerns after releases like DeepSeek’s model.
