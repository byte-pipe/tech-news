# I Am Happier Writing Code by Hand

* The author initially experienced feelings of depression and lethargy when using Claude-code, an LLM for generating code, finding it led to passive acceptance and a lack of deep thinking.
* The author argues that for software engineering, the process of writing code is essential for understanding the problem space and ensuring correctness, as it requires active engagement and internalization.
* "Vibe coding," where code is generated with minimal author input, interferes with this process by allowing the author to skip critical thinking and potentially leading to incorrect or poorly understood code.
* The author describes vibe coding as addictive but also as a way to avoid mental effort, even resorting to simple tasks like find-and-replace being outsourced to the LLM.
* While acknowledging the potential benefits of LLMs, the author now uses Claude-code in a more controlled manner, providing context and asking for specific changes rather than full code generation.
* This deliberate approach maintains engagement, ensures better understanding of the codebase, and prevents the LLM from becoming a passive tool that hinders deep thinking.
* The author concludes that prioritizing happiness in work is important and encourages readers to consider alternative approaches to using LLMs in software engineering, rather than blindly adopting them for potential productivity gains at the cost of cognitive engagement.
