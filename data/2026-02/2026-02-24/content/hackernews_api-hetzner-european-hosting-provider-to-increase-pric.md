[–]Icedman812 points3 points4 points4 hours ago(0 children)

Having used the hardware (specifically AMD servers) in the past, nothing wrong with it. The CPU cores are the same as the server version (3900x vs EPYC 7302 for example), core counts differ - and obviously the memory bandwidth.

Hosts can be (and often are, check the auction servers for configuration for example) outfitted with ECC RAM, so the only thing that does have a downside is the lack of IPMI/OOB management for the boxes. Yeah, you can remote shutdown and so on, but in order to get console level host access, you need to order the KVM for the couple of hours it's available for free.

Support works great, I had one physical host fail in the two years I ran stuff there (never got the information what the problem was with that hardware), so I put in a ticket and the host got replaced after the second ticket. Yeah, had to fix the Proxmox configuration by hand (MAC addresses and I think NIC HWID), but after that was done, everything ran smooth as butter.

Oh yeah, then there's the VLAN thing that I consider a bit of a downside. If you're running VLANs (they're typically on the same NIC) between hosts (I did), you need to make sure the MTU values get corrected. All of this is documented in their Wiki (which is excellent IMHO).
