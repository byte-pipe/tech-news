# New Repository Settings for Configuring Pull Request Access

This changelog announces new settings in GitHub that provide more control over pull request access for repository maintainers.

## Key Changes:

*   **Disable Pull Requests Entirely:** Maintainers can now disable pull requests for their repositories. This hides the pull requests tab, preventing users from viewing or creating new pull requests. This is useful for mirror repositories, read-only codebases, or projects where contributions are managed outside of pull requests.
*   **Restrict Pull Requests to Collaborators:** Maintainers can restrict pull request creation to only collaborators (users with write access). While the pull requests tab remains visible, only collaborators can create new pull requests. This allows for tighter control over who submits changes, particularly during critical development phases.

## Availability:

These settings are available for all public and private repositories and can be configured in Settings > General > Features.

## Mobile App:

Full UI changes for these settings are coming soon to the mobile app. Currently, disabling pull requests will still show the tab, but users won't be able to create new pull requests. Other settings remain consistent across platforms.

## Additional Note:

Temporary interaction limits can still be used to limit activity from specific users on public repositories.
