# AI-built app on Lovable exposed 18K users, researcher claims • The Register

# Summary of “AI‑built app on Lovable exposed 18K users, researcher claims”

## Incident overview
- Tech entrepreneur Taimur Khan discovered 16 security flaws (six critical) in an app hosted on the vibe‑coding platform **Lovable**.
- The app, built with AI‑generated code and powered by Supabase, was featured on Lovable’s Discover page with >100 k views and ~400 up‑votes.
- Khan reported the findings to Lovable, but the support ticket was closed without a response, prompting public disclosure.

## Vulnerability specifics
- All Lovable‑generated apps use Supabase for authentication, file storage, and real‑time updates via PostgreSQL.
- Missing implementation of Supabase’s row‑level security and role‑based access controls caused malformed authentication functions.
- A logic inversion made the guard block legitimate logged‑in users while allowing unauthenticated access, a mistake a human reviewer would spot instantly.
- Similar flawed access‑control logic appeared across multiple critical functions, enabling unauthenticated attackers to:
  - Read any user record
  - Send bulk emails through the platform
  - Delete user accounts
  - Grade student submissions
  - Access admin email addresses

## Impact on users
- Total exposed records: **18,697**
  - Unique email addresses: **14,928**
  - Student accounts (all with emails): **4,538**
  - Enterprise users: **10,505**
  - Full personally‑identifiable information exposed: **870**
- Affected user base included teachers, students from top US universities (e.g., UC Berkeley, UC Davis), and K‑12 institutions with minors.

## Responsibility debate
- Khan argues that Lovable markets itself as a platform delivering “production‑ready apps with authentication included,” so it should ensure the security posture of hosted apps.
- He criticises the lack of a proper response and the absence of mandatory security scans before showcasing apps.
- Lovable’s CISO Igor Andriushchenko counters that:
  - A free security scan is run on every project before publishing, providing remediation recommendations.
  - Implementation of those recommendations is the user’s responsibility.
  - The vulnerable database was not hosted by Lovable, and part of the code was not generated by their tool.
  - The app owner has been contacted and is addressing the issue.
