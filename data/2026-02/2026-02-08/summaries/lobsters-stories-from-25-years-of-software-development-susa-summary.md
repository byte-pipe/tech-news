## Viewing the Source
In 2001, a university student showed the author how to create a website using HTML. The student explained that websites are made of HTML pages, which are text instructions. He demonstrated basic HTML features like changing font, size, centering text, and background color. After the brief tutorial, the student took the seat back and the author was left with newfound knowledge but without a website domain. This encounter sparked a lifelong interest in creating and maintaining personal websites.

## The Reset Vector
During university, while working on an assembly language program for a lift control system on an MS-DOS machine, the author wondered if the system would reboot if the execution jumped to the reset vector address (FFFF:0000). Testing this with DEBUG.EXE confirmed the reboot. A high-achieving classmate witnessed this and expressed a desire to think more like the author – by experimenting with concepts rather than solely focusing on achieving top grades. This incident led the classmate to intentionally stop striving for the top rank in the class to prioritize exploration and learning.

## Man in the Middle
The author's first job involved deploying an e-banking component using a fragile Python installer. The installer frequently failed due to incorrect assumptions about the environment and required manual intervention. The author spent the first few weeks stabilizing the installer and creating a user guide, which received more praise than the technical improvements. Seeking more substantial work, the author eventually transferred to a different city to join a team called Architechture, which developed and maintained a web framework.
