# It's time for modern CSS to kill the SPA - Jono Alderson

This article discusses an important opportunity for solo developers in the web development space. Here's a 3-4 paragraph analysis from a solo developer business perspective:

The key problem being addressed is the overuse of Single Page Application (SPA) architecture for building websites, when a simpler Multi-Page Application (MPA) approach with modern CSS features can often deliver a better user experience and performance. The author argues that the assumption that "seamless navigation requires us to build an app" is now obsolete, as native CSS features like View Transitions API and Speculation Rules can provide smooth, fast page transitions without the complexity of a full SPA.

There are clear market indicators suggesting this is a significant opportunity. The author cites performance metrics showing that a modern MPA approach can outperform a typical Next.js SPA, with much smaller JavaScript bundles and faster Time to Interactive. This points to real user pain points around bloated, sluggish website performance that businesses are likely willing to pay to solve.

From a solo developer perspective, this seems highly technically feasible. The core technologies discussed, like View Transitions and Speculation Rules, are native browser features that don't require complex frameworks or large codebases. A skilled CSS developer could likely implement these techniques relatively quickly, without the time investment needed to build a full SPA. And since the approach is focused on simplicity and performance, it aligns well with the strengths of a solo practitioner.

In terms of business viability, the pain points around SPA performance are widespread, and there appears to be limited competition in the market for MPA-focused web development services. The author also notes that these modern CSS techniques play well with SEO and browser optimization features like the Back/Forward Cache, further enhancing the value proposition. Overall, this seems like a promising opportunity for a solo developer to differentiate themselves and build a profitable business solving a real, underserved market need.
