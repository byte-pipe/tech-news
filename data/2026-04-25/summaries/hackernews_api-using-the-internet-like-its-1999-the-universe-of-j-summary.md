# Using the internet like it’s 1999 – The Universe of Joshua Blais  

## Overview  
- Modern web life is dominated by algorithmic feeds, echo chambers, and AI‑generated noise; this represents only about 3‑5 % of what the internet could offer.  
- The author argues that we can reclaim the original, open, protocol‑driven internet by bypassing modern front‑ends and using older, transparent technologies.  

## 1999 Perspective  
- In 1999 there was no social media, no recommendation algorithms, and Google was just beginning; only ~4 % of the world was online.  
- The goal is not to log off completely but to become “citizens of the web” rather than data points in a corporate database, regaining agency over attention.  

## Core Recommendations  

### RSS Feeds  
- Adopt RSS as the primary way to follow blogs, news, and video creators, avoiding algorithmic curation.  
- Suggested tool: **Miniflux** – add feeds you trust and let content arrive on its own.  

### IRC and XMPP  
- IRC provides low‑signal‑to‑noise, plain‑text chat; XMPP adds modern features and end‑to‑end encryption (OMEMO).  
- Self‑hosted instances give privacy; the author recommends avoiding Matrix due to unnecessary complexity.  

### Search Engines  
- Use personal, intentional search queries rather than default “trending” searches.  
- Example: query with date constraints like `before:<2025> net/http go language` to obtain precise results.  

### Archiving  
- Counter link rot by downloading useful content locally and syncing across devices (e.g., via Syncthing).  
- The Internet Archive’s “Save Page Now” tool can create permanent backups.  

### Email  
- Prefer direct email communication over platform DMs; email is not harvested for ad targeting.  
- Secure messages with PGP encryption; the author provides a public key for contacts.  

### Push‑Only (POSSE) Method  
- Treat social media as a distribution channel only: publish content on your own site first, then push it outward.  
- This follows the IndieWeb POSSE principle, keeping ownership of the original material.  

### Gopher / Gemini  
- Explore lightweight, text‑focused protocols (Gopher, Gemini) for a minimalist browsing experience that aligns with the 1990s ethos.  

## Conclusion  
- By returning to protocol‑level tools—RSS, IRC/XMPP, intentional search, personal archiving, encrypted email, and push‑only publishing—we can restore the internet’s original promise of open, user‑controlled information and escape the modern, advertisement‑driven, algorithmic landscape.
