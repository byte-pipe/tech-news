# Why I’m Done Making Desktop Applications

## Background
- For three years I sold **Bingo Card Creator**, a Swing‑based desktop app that grew from a hobby into a modest business.  
- I defended desktop software against web‑app critics while defending my own product.

## What Changed My Mind
- My day job shifted from thick‑client Swing to large‑scale enterprise web applications (SQL, Rails).  
- I built a web version of Bingo Card Creator this summer; it outperformed the desktop version in development speed, features, sales, support load, and marketability.  
- From a developer’s viewpoint the web app strictly dominates the desktop app.

## The Shareware Funnel Is Lethal
- The classic “download‑trial‑purchase” flow involves about 17 steps (search, download, locate file, install, navigate installer screens, run program, find purchase page, enter payment).  
- Each step is a potential point of failure, leading to typical conversion rates of 1 % or less for shareware.

## Web Applications Convert Better
- No download or installation, no need to restart, and purchasing is seamless.  
- Conversion statistics from my best AdWords campaign:  

  | Metric | Desktop (download) | Web app |
  |--------|-------------------|---------|
  | Visitor → Free trial | 18 % – 22 % | 22 % – 26 % |
  | Trial → Purchase | 1.35 % | 2.32 % |

- Despite having fewer features and less development time, the web version outsells the desktop version roughly two‑to‑one.

## AdWords Strategy Is Very Sensitive to Conversion Rates
- Cost per acquisition (CPA) calculations:  

  - Desktop version: about $20 CPA  
  - Web app: about $9 CPA  

- Lower CPA lets me bid higher on keywords, out‑compete rivals who sell downloadable software, and capture more traffic (≈ 25 % more inventory).  
- The cheaper acquisition cost also enables profitable advertising on high‑volume terms like “bingo cards.”

## Web Applications Are Easier to Support
- Support tickets from the last 50 customers:  

  - Desktop app: 15 tickets  
  - Web app: 3 tickets  

- Reasons for reduced support: no installation issues, no registration keys (single‑sign‑on via e‑junkie and cookies), and users always run the latest version (no outdated copies circulating on download sites).

## Piracy Perspective
- I have been relaxed about piracy, focusing on delivering a good experience to paying customers.  
- Nevertheless, the existence of pirates remains a concern, especially when institutions obtain cracked copies.  

---  

In summary, the web‑based model eliminates the cumbersome shareware funnel, yields higher conversion and lower advertising costs, dramatically reduces support overhead, and offers a more sustainable business compared with traditional desktop applications. Consequently, I have decided to stop creating desktop software and focus entirely on web applications.
