# The Cult Of Vibe Coding Is Insane

## The Problem with "Vibe Coding"

Bram Cohen explains that the concept of "vibe coding" is a myth perpetuated by dogfooding software development. He argues that developers engage in vibe coding without adding value to the project, leading to an environment where bugs and duplication are rampant.

## Key Points:

* Dogfooding involves using own product without making actual contributions.
* Vibe coding is when developers "make no contribution" to the code, essentially being a cult-like behavior where they ignore underlying infrastructure features like TODO lists.
* Despite human involvement, vibe coding leads to significant technical debt and unnecessary duplication of effort.

## The Consequences:

* Human error often manifests in duplicate code or inefficient infrastructure:
	+ Duplication between developers raises questions: why didn't any developer inspect the code before adding their own?
* This process is repeated over time, with no regard for clean-up efforts:
	+ Traditional software development processes often leave significant tech debt that's difficult to recover.

## The Solution:

* Developers should strive to maintain high-quality software by actively improving and cleaning up messes.
* AI algorithms can assist in identifying unnecessary duplication and suggesting ways to improve infrastructure, allowing developers to focus on meaningful contributions.
