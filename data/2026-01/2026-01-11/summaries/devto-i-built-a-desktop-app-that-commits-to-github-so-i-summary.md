# I Built a Desktop App That Commits to GitHub So I Don’t Have To Lie About Consistency - DEV Community

## Introduction to a Desktop App Automating Commit Consistency
### Main Idea
The author built a desktop app that automates commits for productivity and reduces mental overhead of "oh no I forgot to commit today". This app was developed in response to the common misconception that GitHub contribution graphs are a productivity metric, but not.

## Key Concepts and Benefits
- **Consistency**: Not just discipline, but systems for achieving consistency.
- **Automation**: Removing manual efforts and making the process more efficient.
- **Simplification**: Creating an easy-to-use tool with minimal setup required.
- **Reduction of "no commit" friction**: Automating commits saves time and reduces mental effort.

## Implementation Details
- **Running Locally**: The app can be installed locally, with schedule commands, no need for browser tabs or cron jobs.
- **Automating Jobs**: Pushing commits directly to the repository eliminates the need for a separate development environment.
- **Open Configuration**: Simply configuring the app once and it will handle the rest.

## Implications and Takeaways
This application was created by addressing inconsistencies in code writing, which has become apparent through experience. The project illustrates the importance of automating repetitive tasks to enhance productivity.

## Related Questions or Thoughts
The introduction encourages a discussion about whether "fake productivity metrics" reveal our actual discipline in achieving goals rather than just assuming them is essential.
