# Users Cannot Create Issues Directly in Ghostty

**Overview**

The current implementation of issues creation in the Ghostty repository does not allow direct creation by users without creating a discussion first. This approach differs from other projects that use the issue tracker for discussions and feature requests.

**Current Problem**

In addition to being unable to create issues directly, users often experience common problems such as bugs (80-90% of what they consider) or configuration errors due to misunderstandings and environmental factors. However, feature requests account for around 70% and underspecified requirements need guidance from maintainers.

## Key Points

* Users are not allowed direct issue creation in Ghostty.
* Issue tracker is used instead for discussion and feature requests.
* Pattern of using GitHub discussions before creating issues helps determine relevance.
* Direct issue creation bypasses user effort, only necessary when a well-defined problem identification process yields an actionable item.
