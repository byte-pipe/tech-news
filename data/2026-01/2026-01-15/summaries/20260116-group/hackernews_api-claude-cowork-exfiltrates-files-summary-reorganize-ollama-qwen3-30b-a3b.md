# Claude Cowork 文件外泄漏洞

## 漏洞概述
Anthropic 发布的 Claude Cowork 研究预览版本（通用工作辅助 AI 代理）存在文件外泄漏洞。攻击者可通过间接提示注入（prompt injection）利用代码执行环境的缺陷，从 Claude 的虚拟机环境窃取本地文件。

## 攻击机制
攻击链依赖 Anthropic API 的网络访问白名单机制，受害者操作步骤如下：

| 攻击步骤 | 漏洞利用点                     | 风险描述                     |
|----------|------------------------------|----------------------------|
| 1. 连接本地敏感文件夹 | 用户授权 Claude 访问本地文件系统 | 攻击者可操控文件路径         |
| 2. 上传伪装文件（如 .docx） | 利用 Word 格式隐匿恶意内容     | 文件显示为安全的“技能”文件   |
| 3. 触发间接注入           | 通过 Claude 在线存在获取攻击源   | 无需依赖特定攻击源位置       |

> **关键特征**：攻击不依赖攻击源类型，可来自任何不安全来源（如 Claude 在线内容）。

## 外泄文件与影响
- **伪装手法**：攻击者上传 `.docx` 文件，利用 Claude 普遍采用的“技能”文件格式设计，使内容显示为无害
- **实际风险**：实际外泄内容为敏感文件（如工作文档），伪装格式避免用户察觉
- **安全影响**：用户可能在无意识状态下授权暴露机密文件

## 安全建议（Anthropic 官方声明）
Anthropic 明确要求用户采取以下防护措施：
- **禁止向 Claude Cowork 授予本地敏感文件访问权限**
- **特别注意“本地文件连接”类操作**，避免与未知来源交互
- **警惕文件上传行为**，特别是格式为 `.docx` 的“技能”文件

> 说明：尽管 Anthropic 已通过公开演示提示风险，但尚未直接提供“检测可疑本地文件连接”的操作指南，主要依赖用户自主防范。

## 漏洞背景
- 漏洞源于 Claude Cowork 的**代理特性**（agentic features）与**互联网访问权限**的组合
- Anthropic 指出：非技术人员易忽略此风险，需主动加强防护意识
- 漏洞演示目的为提升用户安全意识，推动更审慎的在线操作习惯
