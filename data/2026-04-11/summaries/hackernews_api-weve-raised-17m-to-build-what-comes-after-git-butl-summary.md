# We’ve raised $17M to build what comes after Git

## Announcement & Funding
- GitButler announced a $17 million Series A round led by a16z, with continued participation from Fly Ventures and A Capital.  
- Peter Levine (a16z) joins the board; he previously served with Scott Chacon on GitHub’s board.

## Founder’s Perspective
- Scott Chacon, co‑founder of GitHub, reflects on 15 years of watching Git evolve from a niche tool to the core of global software development.  
- He notes that developer platforms succeed when they eliminate friction in collaboration and reduce overhead for code creators.

## Vision for GitButler
- The current development workflow has been forced to fit Git’s historic constraints; GitButler aims to create tooling designed for modern practices.  
- The company believes software development is entering a new phase where the problems Git solved for the past two decades need a redesign, especially given the rise of AI‑driven agents.

## Problems with the Existing Git Model
- Context disintegrates across tools, people, and now AI agents.  
- The traditional “one person, one branch, one terminal, linear flow” model is insufficient for stacked branches, multitasking, and AI integration.

## GitButler CLI Technical Preview
- A CLI built for GitHub Flow‑style, short‑lived branch, trunk‑based workflows.  
- Features include branch stacking, multitasking, easy undo, and compatibility with humans, agents, and scripts.  
- It can be added to any existing Git project without disruption.

## Toward Social Coding
- Current collaboration still relies on separate issue lists, Kanban boards, external chats, and PRs that lose context.  
- Desired capabilities:
  * Automatic crafting of logical, context‑rich changes.  
  * Visibility of agent interactions and related conversations.  
  * Early detection of merge conflicts between teammates.  
  * Ability to work on branches stacked on a coworker’s branch while both modify them.  
  * Real‑time awareness of what every team member and AI agent is doing.

## Mission Statement
- GitButler is not trying to build a “better Git”; it is building the next‑generation infrastructure for how software will be built.

## Author Bio
- Scott Chacon is a co‑founder of GitHub and GitButler, author of *Pro Git*, and a global speaker on Git and software collaboration.
